= System Notifier

A collection of system monitoring services that send desktop notifications for important system events.

== What this is

System Notifier provides monitoring services for:

* Battery levels
* Disk space usage

Users receive desktop notifications when thresholds are crossed. Notifications remain on screen until dismissed.

== Architecture

This is a Rust-based monorepo containing multiple systemd user services that share common notification code.

=== Components

==== Battery monitor service

Monitors battery levels and sends notifications at three thresholds (defaults):

* 15% - Low battery notice
* 10% - Low battery warning
* 5% - Critical battery warning (with shutdown advisory)

==== Disk space monitor service

Monitors disk space usage for configured paths and sends notifications at three thresholds (defaults):

* 90% - Low disk space notice
* 95% - Low disk space warning
* 97.5% - Disk space critical

==== Notification library

Shared library for sending desktop notifications using the freedesktop notification system.

==== Config system

Both monitors load user configuration from the first file found at these paths (in order):

. `~/.config/system-monitor.toml`
. `~/.config/system-monitor/config.toml`
. `~/.system-monitor.toml`

Any values not specified in the config file are filled in from built-in defaults.

== Installation

=== Building the Debian package

Build the Debian package:

[source,bash]
----
dpkg-buildpackage -us -uc -b
----

The generated `.deb` file appears in the parent directory.

=== Installing the package

Install the package:

[source,bash]
----
sudo dpkg -i ../system-notifier_*.deb
----

=== Enabling the services

The timers are enabled automatically on installation. To manually control them:

Enable and start battery monitor:

[source,bash]
----
systemctl --user enable --now battery-monitor.timer
----

Enable and start disk space monitor:

[source,bash]
----
systemctl --user enable --now diskspace-monitor.timer
----

=== Checking service status

Check the status of the timers:

[source,bash]
----
systemctl --user status battery-monitor.timer
systemctl --user status diskspace-monitor.timer
----

View recent log entries:

[source,bash]
----
journalctl --user -u battery-monitor.service
journalctl --user -u diskspace-monitor.service
----

== Configuration

Both monitors are configured through a TOML file. The first file found at these locations is used:

. `~/.config/system-monitor.toml`
. `~/.config/system-monitor/config.toml`
. `~/.system-monitor.toml`

Any option not present in the file falls back to the built-in default. The file does not need to be complete.

=== Example configuration

[source,toml]
----
[battery]
disabled = false

[battery.thresholds]
low = 20.0        # default: 15.0
very_low = 12.0   # default: 10.0
critical = 7.0    # default: 5.0

[diskspace]
paths = ["/", "/home", "/data"]

[diskspace.thresholds]
low = 85.0        # default: 90.0
very_low = 92.0   # default: 95.0
critical = 96.0   # default: 97.5
----

=== Disabling a monitor

NOTE:: This does not disable the systemd timer, so the service will still run at the configured interval but exit immediately without performing any checks.

Set `disabled = true` in the relevant section to skip all checks for that monitor without disabling the systemd timer:

[source,toml]
----
[battery]
disabled = true
----

=== Monitoring frequency

Both services run every 5 minutes by default. To change this, edit the timer files in `/usr/lib/systemd/user/` and reload systemd:

[source,bash]
----
systemctl --user daemon-reload
systemctl --user restart battery-monitor.timer
systemctl --user restart diskspace-monitor.timer
----

=== Monitored paths

The disk space monitor checks `/` by default. Override this with the `paths` key:

[source,toml]
----
[diskspace]
paths = ["/", "/home"]
----

== Troubleshooting

=== Notifications not appearing

Check that the notification daemon is running:

[source,bash]
----
ps aux | grep notification
----

Test notifications manually:

[source,bash]
----
notify-send "Test" "This is a test notification"
----

=== Service not running

Check service status:

[source,bash]
----
systemctl --user status battery-monitor.timer
systemctl --user status diskspace-monitor.timer
----

View service logs:

[source,bash]
----
journalctl --user -xeu battery-monitor.service
journalctl --user -xeu diskspace-monitor.service
----

=== Config file is malformed

If the config file cannot be parsed, both monitors send a desktop notification and print the error to stderr, then exit without checking. Fix the config file, then check logs for the parse error:

[source,bash]
----
journalctl --user -u battery-monitor.service
journalctl --user -u diskspace-monitor.service
----

=== Battery service fails on desktop systems

The battery monitor exits silently on systems without a battery. You can disable the battery monitor timer on desktop systems:

[source,bash]
----
systemctl --user disable --now battery-monitor.timer
----

Or set `disabled = true` in the config file under `[battery]`.
